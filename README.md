# Python 基础学习路线

## 这条路线是什么？

这是一条 **Python 基础知识** 的完整学习路线，涵盖从环境搭建到面向对象、文件操作、异常处理、模块化，再到迭代器和常用进阶语法的所有核心内容。

这条路线按照 **环境搭建 → 数据类型 → 流程控制 → 函数 → 面向对象 → 文件与异常 → 模块化 → 进阶特性** 的顺序编排，每一步都是下一步的地基。

### 适合谁？

- 零基础想学 Python 的同学
- 有其他语言基础，想快速上手 Python 的同学
- 基础不扎实，想系统补一遍的同学

## 学习路线

### 🟢 第一阶段：环境与基础（预计 2-3 天）

**目标：搭好开发环境，能写出第一行 Python 代码并运行。**

> 💡 工欲善其事，必先利其器。环境搞不好，后面全白搭。

1. **Python 安装与环境配置** ⭐⭐⭐ 🎯
   - 下载安装 Python（建议 3.10+）
   - 配置环境变量（Windows 安装时勾选 Add to PATH）
   - 验证安装：命令行输入 `python --version`
   - 「一句话总结」：装好 Python，能在命令行跑起来，就算成功
   - ⚠️ 踩坑：Windows 用户安装时一定要勾选"Add Python to PATH"，否则命令行找不到 python 命令

2. **IDE 选择** ⭐⭐⭐
   - **PyCharm**：功能强大，适合大型项目，社区版免费
   - **VS Code**：轻量灵活，装上 Python 插件就能用
   - 「一句话总结」：新手推荐 PyCharm 社区版，开箱即用；喜欢轻量的选 VS Code

3. **pip 包管理** ⭐⭐⭐
   - `pip install 包名`：安装第三方包
   - `pip list`：查看已安装的包
   - `pip freeze > requirements.txt`：导出依赖
   - `pip install -r requirements.txt`：批量安装依赖
   - 换源加速：`pip install -i https://pypi.tuna.tsinghua.edu.cn/simple 包名`

4. **虚拟环境** ⭐⭐⭐
   - `python -m venv venv` 创建虚拟环境
   - 激活虚拟环境：Windows `venv\Scripts\activate`，Mac/Linux `source venv/bin/activate`
   - 为什么要用：给每个项目一个独立的"沙盒"，互不干扰
   - 了解 Poetry / PDM 等现代包管理工具
   - 「一句话总结」：虚拟环境 = 项目专属的 Python 环境，不同项目的依赖互不打架
   - ⚠️ 踩坑：不用虚拟环境，所有包全局安装，项目 A 要 requests 2.28，项目 B 要 requests 2.31，直接炸

5. **代码注释与编码规范** ⭐⭐
   - 单行注释：`# 这是注释`
   - 多行注释：`'''这是多行注释'''` 或 `"""这也是"""`
   - PEP 8 编码规范：缩进用 4 个空格、变量名用小写下划线、类名用大驼峰
   - 「一句话总结」：写代码是给人看的，顺便让机器执行

### 🔵 第二阶段：数据类型与运算符（预计 3-4 天）

**目标：掌握 Python 所有基本数据类型和运算操作，能灵活处理各种数据。**

> 💡 数据类型是编程的"原材料"。你得先认识这些材料，才能用它们搭建程序。

6. **基本数据类型** ⭐⭐⭐ 🎯
   - 整数 `int`：`age = 18`
   - 浮点数 `float`：`price = 9.99`
   - 布尔值 `bool`：`True` / `False`
   - 字符串 `str`：`name = "Python"`
   - 空值 `NoneType`：`result = None`
   - 「一句话总结」：Python 的数据类型就这几种，记住它们是一切的起点

7. **字符串操作** ⭐⭐⭐ 🎯
   - 切片：`s[1:5]`、`s[::-1]`（反转）
   - 格式化：f-string `f"我叫{name}，今年{age}岁"`、`.format()`
   - 常用方法：`split`、`join`、`strip`、`replace`、`find`、`startswith`、`endswith`、`upper`、`lower`
   - 「一句话总结」：字符串是用得最多的数据类型，这些方法必须熟练
   - ⚠️ 踩坑：字符串是不可变类型，`s.replace("a", "b")` 不会修改原字符串，而是返回新字符串

8. **容器类型** ⭐⭐⭐ 🎯
   - **列表 list**：有序、可变、可重复，`[1, 2, 3]`，增删改查、排序、切片
   - **元组 tuple**：有序、不可变、可重复，`(1, 2, 3)`，适合存不需要修改的数据
   - **字典 dict**：键值对、无序（3.7+ 有序）、键唯一，`{"name": "张三", "age": 18}`
   - **集合 set**：无序、不重复，`{1, 2, 3}`，适合去重和集合运算
   - 「一句话总结」：list 最常用，dict 第二常用，tuple 用来存不变的数据，set 用来去重

9. **类型转换与类型判断** ⭐⭐
   - 类型转换：`int()`、`str()`、`float()`、`list()`、`tuple()`、`set()`、`dict()`
   - 类型判断：`type(x)` 查看类型、`isinstance(x, int)` 判断是否是某类型
   - 「一句话总结」：Python 是动态类型语言，类型转换是家常便饭

10. **运算符** ⭐⭐⭐ 🎯
    - **算术运算符**：`+`、`-`、`*`、`/`（真除）、`//`（整除）、`%`（取余）、`**`（幂）
    - **比较运算符**：`==`、`!=`、`>`、`<`、`>=`、`<=`
    - **逻辑运算符**：`and`、`or`、`not`
    - **赋值运算符**：`=`、`+=`、`-=`、`*=`、`/=` 等
    - **成员运算符**：`in`、`not in`（判断元素是否在容器中）
    - **身份运算符**：`is`、`is not`（判断是否是同一个对象）
    - **位运算符**：`&`、`|`、`^`、`~`、`<<`、`>>`（了解即可）
    - **三元表达式**：`x = a if 条件 else b`
    - 「一句话总结」：运算符就是对数据做操作的"动词"
    - ⚠️ 踩坑：`==` 比较值是否相等，`is` 比较是否是同一个对象，别搞混了

### 🟡 第三阶段：流程控制（预计 3-4 天）

**目标：掌握条件判断和循环，能用代码表达逻辑。**

> 💡 流程控制是编程的"骨架"。有了它，代码才能根据不同情况做不同的事。

11. **条件判断** ⭐⭐⭐ 🎯
    - `if / elif / else`
    - 嵌套条件判断
    - 条件表达式（三元表达式）：`result = "成年" if age >= 18 else "未成年"`
    - 「一句话总结」：`if age > 18: print("成年了")` 一看就懂，这就是 Python 的魅力

12. **循环** ⭐⭐⭐ 🎯
    - `for` 循环：遍历列表、字典、字符串、range()
    - `while` 循环：条件为真就一直执行
    - `break`：跳出整个循环
    - `continue`：跳过本次循环，进入下一次
    - `pass`：占位符，什么都不做
    - `range()` 函数：`range(10)`、`range(1, 10)`、`range(0, 10, 2)`
    - 循环嵌套：循环里面套循环
    - `match-case`（Python 3.10+）：类似其他语言的 switch-case
    - 「一句话总结」：for 用来遍历，while 用来重复，break 用来跳出
    - ⚠️ 踩坑：Python 用缩进表示代码块，Tab 和空格混用会报错，而且报错信息经常不指向真正出问题的那一行

13. **推导式** ⭐⭐⭐ 🎯
    - **列表推导式**：`[x * 2 for x in range(10) if x > 3]`
    - **字典推导式**：`{k: v for k, v in items}`
    - **集合推导式**：`{x for x in list1}`
    - **生成器表达式**：`(x * 2 for x in range(10))`（惰性求值，省内存）
    - 「一句话总结」：推导式是 Python 的语法糖，一行代码顶三行

### 🟠 第四阶段：函数（预计 4-5 天）

**目标：掌握函数的定义和使用，能写出结构清晰、可复用的代码。**

> 💡 函数是代码复用的基本单位。把重复的逻辑封装成函数，代码立刻变得清晰。

14. **函数定义与调用** ⭐⭐⭐ 🎯
    - `def` 定义函数
    - 参数类型：
      - 位置参数：`def add(a, b)`
      - 默认参数：`def greet(name="世界")`
      - 可变参数：`def func(*args)`（接收任意个位置参数，打包成元组）
      - 关键字参数：`def func(**kwargs)`（接收任意个关键字参数，打包成字典）
    - `return` 返回值，支持多值返回：`return a, b`（本质是返回元组）
    - 「一句话总结」：函数 = 给一段代码起个名字，需要的时候喊一声就行
    - ⚠️ 踩坑：默认参数不要用可变对象（如 list、dict），否则多次调用会共享同一个对象

15. **作用域** ⭐⭐⭐
    - 局部变量（local）：函数内部定义的变量
    - 全局变量（global）：函数外部定义的变量
    - `global` 关键字：在函数内修改全局变量
    - `nonlocal` 关键字：在嵌套函数中修改外层函数的变量
    - LEGB 规则：Local → Enclosing → Global → Built-in
    - 「一句话总结」：变量的"可见范围"就是作用域，搞不清楚就会出 bug

16. **lambda 匿名函数** ⭐⭐
    - 语法：`lambda x, y: x + y`
    - 常配合 `map`、`filter`、`sorted` 使用
    - 「一句话总结」：lambda 就是"一次性的小函数"，用完就扔

17. **递归函数** ⭐⭐
    - 函数调用自身
    - 必须有终止条件，否则无限递归
    - 经典例子：阶乘、斐波那契数列
    - 「一句话总结」：递归 = 自己调用自己 + 终止条件
    - ⚠️ 踩坑：Python 默认递归深度限制 1000 层，超过会报 RecursionError

18. **闭包** ⭐⭐⭐
    - 内层函数引用了外层函数的变量，并且外层函数返回了内层函数
    - 闭包可以"记住"外层函数的变量
    - 「一句话总结」：闭包 = 函数 + 它记住的环境变量

19. **装饰器（decorator）** ⭐⭐⭐ 🎯
    - 理解闭包是装饰器的基础
    - `@` 语法糖
    - 不带参数的装饰器
    - 带参数的装饰器
    - `functools.wraps` 保留原函数信息
    - 「一句话总结」：装饰器 = 不修改原函数代码，给它"加功能"
    - ⚠️ 面试高频：装饰器的原理和手写装饰器，几乎每场 Python 面试都会问

20. **类型注解（Type Hints）** ⭐⭐⭐
    - 函数参数和返回值类型标注：`def add(a: int, b: int) -> int:`
    - `typing` 模块：`List`、`Dict`、`Optional`、`Union`、`Tuple`
    - 类型注解不影响运行，但能提高代码可读性和 IDE 提示
    - 「一句话总结」：类型注解是给人和 IDE 看的"说明书"，Python 本身不强制检查

21. **常用内置函数** ⭐⭐⭐
    - `len`、`max`、`min`、`sum`、`abs`、`round`
    - `zip`：把多个列表"拉链式"配对
    - `map`：对每个元素应用函数
    - `filter`：过滤元素
    - `enumerate`：遍历时同时获取索引和值
    - `sorted`：排序（支持 key 参数自定义排序规则）
    - `any`、`all`：判断是否有/全部为真
    - `isinstance`、`type`：类型判断
    - 「一句话总结」：这些内置函数是 Python 的"瑞士军刀"，用好了代码又短又优雅

### 🔴 第五阶段：面向对象编程 OOP（预计 5-6 天）

**目标：掌握面向对象的核心思想，能用类和对象组织代码。**

> 💡 OOP 就是把数据和操作数据的方法打包成一个"对象"。后端框架（FastAPI、Django）全是基于 OOP 的，不学 OOP 等于看不懂框架源码。

22. **类和对象基础** ⭐⭐⭐ 🎯
    - `class` 定义类
    - `__init__` 构造方法：创建对象时自动调用
    - `self` 关键字：指向当前实例
    - 实例属性 vs 类属性
    - 「一句话总结」：类是"图纸"，对象是按图纸造出来的"产品"

23. **方法类型** ⭐⭐⭐
    - 实例方法：`def method(self)`，操作实例数据
    - 类方法：`@classmethod`，`def method(cls)`，操作类级别数据
    - 静态方法：`@staticmethod`，`def method()`，和类相关但不需要访问类或实例
    - `@property` 属性装饰器：把方法变成属性访问
    - 「一句话总结」：三种方法各有用途，实例方法最常用

24. **继承与多态** ⭐⭐⭐ 🎯
    - 单继承：`class Dog(Animal):`
    - 多继承：`class C(A, B):`（了解 MRO 方法解析顺序）
    - `super()` 调用父类方法
    - 方法重写（Override）：子类覆盖父类的方法
    - 多态：不同对象调用同一方法，表现不同
    - 「一句话总结」：继承 = 复用代码，多态 = 同一接口不同实现

25. **封装** ⭐⭐
    - 单下划线 `_name`：约定俗成的"私有"，外部可以访问但不建议
    - 双下划线 `__name`：名称改写（name mangling），外部不能直接访问
    - 「一句话总结」：Python 没有真正的私有，靠的是"君子协定"
    - ⚠️ 踩坑：`__name` 只是名称改写，通过 `_ClassName__name` 还是能访问，别把它当成 Java 的 private

26. **魔术方法（Magic Methods）** ⭐⭐⭐ 🎯
    - `__str__`：print 时的输出
    - `__repr__`：开发调试时的输出
    - `__len__`：让对象支持 `len()`
    - `__eq__`、`__lt__`：让对象支持比较
    - `__add__`：让对象支持 `+` 运算
    - `__getitem__`：让对象支持 `obj[key]` 访问
    - `__iter__`、`__next__`：让对象可迭代
    - `__enter__`、`__exit__`：让对象支持 `with` 语句
    - 「一句话总结」：魔术方法让你的类"融入" Python 的语法体系

27. **dataclass** ⭐⭐
    - `@dataclass` 装饰器：自动生成 `__init__`、`__repr__`、`__eq__` 等方法
    - 减少样板代码，适合纯数据类
    - 「一句话总结」：dataclass 就是"懒人版的类定义"

28. **抽象类** ⭐⭐
    - `ABC`（Abstract Base Class）和 `@abstractmethod`
    - 定义接口规范，子类必须实现抽象方法
    - 「一句话总结」：抽象类 = "规定子类必须实现哪些方法"的模板

### 🟣 第六阶段：文件操作与异常处理（预计 2-3 天）

**目标：能读写文件、处理 JSON 数据，能优雅地处理程序中的错误。**

> 💡 文件操作和异常处理是实际开发中天天用的技能。读配置文件、写日志、处理用户上传的文件，都离不开这些。

29. **文件操作** ⭐⭐⭐ 🎯
    - `open()` 打开文件
    - 文件模式：`r`（读）、`w`（写，覆盖）、`a`（追加）、`rb`（二进制读）、`wb`（二进制写）
    - 读取：`read()`、`readline()`、`readlines()`
    - 写入：`write()`、`writelines()`
    - `with` 上下文管理器：自动关闭文件，不用手动 `close()`
    - 「一句话总结」：读写文件永远用 `with open(...) as f:`，安全又省心

30. **路径操作** ⭐⭐
    - `os` 模块：`os.path.join()`、`os.path.exists()`、`os.listdir()`、`os.makedirs()`
    - `pathlib` 模块（推荐）：面向对象的路径操作，`Path("data") / "file.txt"`
    - 「一句话总结」：路径操作推荐用 `pathlib`，比 `os.path` 更优雅

31. **JSON 读写** ⭐⭐⭐
    - `json.loads()`：JSON 字符串 → Python 字典
    - `json.dumps()`：Python 字典 → JSON 字符串
    - `json.load()` / `json.dump()`：直接读写 JSON 文件
    - `ensure_ascii=False`：让中文正常显示
    - 「一句话总结」：JSON 是前后端数据交换的"通用语言"，必须熟练

32. **CSV 读写** ⭐⭐
    - `csv` 模块：`csv.reader()`、`csv.writer()`、`csv.DictReader()`
    - 「一句话总结」：处理表格数据的基本技能

33. **异常处理** ⭐⭐⭐ 🎯
    - `try / except / else / finally`
    - 常见异常类型：`ValueError`、`TypeError`、`KeyError`、`IndexError`、`FileNotFoundError`、`ZeroDivisionError`
    - `raise` 主动抛出异常
    - 自定义异常类：继承 `Exception`
    - `assert` 断言：调试时使用
    - 「一句话总结」：异常处理 = 给代码加"保险"，出错了不至于整个程序崩掉
    - ⚠️ 踩坑：不要用空的 `except:` 捕获所有异常，这会把真正的 bug 也吞掉，排查问题时两眼一黑

---

### 🔵 第七阶段：模块与包（预计 1-2 天）

**目标：掌握模块化编程，能组织大型项目的代码结构。**

> 💡 当代码超过几百行，就必须拆分成多个文件。模块化是写"工程级代码"的基础。

34. **模块导入** ⭐⭐⭐ 🎯
    - `import 模块名`
    - `from 模块名 import 函数名`
    - `as` 别名：`import numpy as np`
    - `__name__ == '__main__'`：判断是直接运行还是被导入
    - 「一句话总结」：一个 `.py` 文件就是一个模块，`import` 就是"借用"别人的代码

35. **自定义模块与包** ⭐⭐⭐
    - 自定义模块：写一个 `.py` 文件，别的文件 `import` 它
    - 包（Package）：一个包含 `__init__.py` 的文件夹
    - 包的层级结构和导入方式
    - 「一句话总结」：模块是单个文件，包是一组模块的文件夹

36. **常用标准库** ⭐⭐⭐
    - `os`、`sys`：系统交互、路径操作
    - `datetime`、`time`：日期时间处理
    - `re`：正则表达式
    - `math`：数学运算
    - `random`：随机数
    - `collections`：Counter（计数器）、defaultdict（默认字典）、OrderedDict（有序字典）
    - `itertools`：排列组合、无限迭代器
    - `functools`：partial（偏函数）、lru_cache（缓存）、wraps（装饰器辅助）
    - 「一句话总结」：标准库是 Python "自带的工具箱"，不用安装就能用

### 🟡 第八阶段：正则表达式（预计 1-2 天）

**目标：掌握正则表达式的基本语法，能用它做字符串匹配和提取。**

> 💡 正则表达式在数据清洗、表单验证、日志分析中非常常用。不需要精通，但基本语法必须会。

37. **正则表达式（re 模块）** ⭐⭐⭐
    - `re.match()`：从开头匹配
    - `re.search()`：搜索第一个匹配
    - `re.findall()`：找出所有匹配
    - `re.sub()`：替换匹配内容
    - 常用语法：
      - `\d`（数字）、`\w`（字母数字下划线）、`\s`（空白字符）
      - `.`（任意字符）、`*`（0 次或多次）、`+`（1 次或多次）、`?`（0 次或 1 次）
      - `[]`（字符集）、`()`（分组）、`^`（开头）、`$`（结尾）
    - 「一句话总结」：正则表达式是"文本处理的瑞士军刀"，学会了处理字符串效率翻倍
    - ⚠️ 踩坑：正则写复杂了自己都看不懂，建议加注释或用 `re.VERBOSE` 模式

---

### 🔴 第九阶段：迭代器与生成器（预计 2-3 天）

**目标：理解 Python 的迭代协议，掌握生成器的使用。**

> 💡 迭代器和生成器是 Python 的精华特性，也是面试高频考点。理解了它们，你对 Python 的理解会上一个台阶。

38. **可迭代对象与迭代器** ⭐⭐⭐ 🎯
    - 可迭代对象（Iterable）：实现了 `__iter__` 方法的对象（list、dict、str 等）
    - 迭代器（Iterator）：实现了 `__iter__` 和 `__next__` 方法的对象
    - `for` 循环的本质：调用 `__iter__` 获取迭代器，然后不断调用 `__next__`
    - 「一句话总结」：可迭代对象 = "能被 for 循环遍历的东西"

39. **生成器** ⭐⭐⭐ 🎯
    - `yield` 关键字：把函数变成生成器
    - 惰性求值：用到一个才算一个，不会一次性把所有数据加载到内存
    - 生成器表达式：`(x * 2 for x in range(10))`
    - `itertools` 模块：`chain`、`islice`、`count`、`cycle`、`product`、`permutations`、`combinations`
    - 「一句话总结」：生成器 = "按需生产"，处理大数据时能省大量内存
    - ⚠️ 面试高频：生成器和列表的区别、yield 的工作原理

### 🟣 第十阶段：进阶语法（预计 2-3 天）

**目标：掌握 Python 的进阶特性，写出更 Pythonic 的代码。**

> 💡 这些特性是区分"会写 Python"和"写好 Python"的分水岭。面试中也经常考到。

40. **上下文管理器** ⭐⭐⭐
    - `with` 语句的原理：`__enter__` 和 `__exit__`
    - 自定义上下文管理器
    - `contextlib.contextmanager` 装饰器：用生成器实现上下文管理器
    - 「一句话总结」：上下文管理器 = "自动帮你善后"（关文件、释放锁、断开连接）

41. **深拷贝与浅拷贝** ⭐⭐⭐
    - 赋值：`b = a`（只是引用，指向同一个对象）
    - 浅拷贝：`copy.copy()`（只复制第一层，嵌套对象还是共享的）
    - 深拷贝：`copy.deepcopy()`（完全独立的副本）
    - 「一句话总结」：赋值不是复制，浅拷贝只复制一层，深拷贝才是真正的"克隆"
    - ⚠️ 踩坑：列表嵌套列表时，浅拷贝修改内层列表会影响原对象

42. **解包操作** ⭐⭐
    - 序列解包：`a, b, c = [1, 2, 3]`
    - 星号解包：`first, *rest = [1, 2, 3, 4, 5]`
    - 字典解包：`{**dict1, **dict2}`
    - 函数参数解包：`func(*args, **kwargs)`
    - 「一句话总结」：解包让代码更简洁，Python 特有的优雅写法

43. **多线程与多进程（了解）** ⭐⭐
    - GIL（全局解释器锁）：Python 多线程不能真正并行执行 CPU 密集型任务
    - `threading` 多线程：适合 I/O 密集型任务（网络请求、文件读写）
    - `multiprocessing` 多进程：适合 CPU 密集型任务（数据计算）
    - 「一句话总结」：I/O 密集用多线程，CPU 密集用多进程

44. **协程基础（了解）** ⭐⭐
    - `async` / `await` 语法
    - `asyncio` 模块
    - 异步编程的概念：单线程内的"并发"
    - 「一句话总结」：协程是 Python 异步编程的基础，FastAPI 就是基于协程的
    - 💡 这里只需要了解概念，后面学 FastAPI 的时候会深入
